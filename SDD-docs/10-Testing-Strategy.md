## 3. 单元测试

### 3.1 distance.py

| 测试用例 | 输入 | 期望输出 |
|----------|------|----------|
| 同一点距离 | `(22.5, 114.0, 22.5, 114.0)` | `0.0` |
| 已知两点距离 | 北京→上海坐标 | ≈1067km（误差<1%） |
| 小距离精度 | 相距100m的两点 | 误差<1m |
| `check_all_distances` 全通过 | 3个门店，距离<100m | `(True, max_dist)` |
| `check_all_distances` 不通过 | 3个门店，其中一对>200m | `(False, max_dist)` |
| `check_all_distances` 单门店 | 1个门店 | `(True, 0.0)` |
| `calculate_max_distance` | 多个门店 | 正确的最大值 |

### 3.2 amap_api.py — deduplicate_stores

| 测试用例 | 输入 | 期望输出 |
|----------|------|----------|
| 空列表 | `[]` | `[]` |
| 单个门店 | `[store]` | `[store]` |
| 无重复 | 距离远的门店列表 | 原列表不变 |
| 有重复 | 两个距离<50m的门店 | 保留名称最长的 |
| 多组重复 | 3组各2个重复门店 | 每组保留1个 |

### 3.3 cluster_finder.py — _deduplicate_clusters

| 测试用例 | 输入 | 期望输出 |
|----------|------|----------|
| 空列表 | `[]` | `[]` |
| 无重叠 | 门店不重叠的商圈 | 全部保留 |
| 有重叠 | 两个商圈共享一个门店 | 保留 brand_count 多的 |
| brand_count 相同 | 两个相同品牌数的重叠商圈 | 保留 max_distance 小的 |

### 3.4 cluster_finder_optimized.py — SpatialGrid

| 测试用例 | 输入 | 期望输出 |
|----------|------|----------|
| 网格分配 | 已知坐标的门店 | 正确的网格键 |
| 邻域查找 | 距离<阈值的门店 | 在 nearby 集合中 |
| 邻域排除 | 距离>阈值的门店 | 不在 nearby 集合中 |
| 自身排除 | 查找自己的邻域 | 不包含自身索引 |

### 3.5 必选品牌过滤

| 测试用例 | 输入 | 期望输出 |
|----------|------|----------|
| 无必选品牌 | `required_brands=None` | 行为不变 |
| 一个必选 | `required_brands=["A"]` | 所有结果含品牌 A |
| 多个必选 | `required_brands=["A","B"]` | 所有结果同时含 A 和 B |
| 全部必选 | 所有品牌都必选 | 等价于不设必选 |
| min_r 计算 | 3个必选品牌 | `min_r=3` |

## 4. 集成测试

### 4.1 find_clusters 端到端

使用固定的 mock 门店数据，不调用实际 API。

**测试数据设计**：

```python
# 构造一组已知应该能聚合的门店
MOCK_BRAND_STORES = {
    "品牌A": [
        {"name": "A1", "address": "addr", "lat": 22.5400, "lon": 114.0600, "poi_id": "a1", "type": ""},
        {"name": "A2", "address": "addr", "lat": 22.5500, "lon": 114.0700, "poi_id": "a2", "type": ""},
    ],
    "品牌B": [
        {"name": "B1", "address": "addr", "lat": 22.5401, "lon": 114.0601, "poi_id": "b1", "type": ""},
        {"name": "B2", "address": "addr", "lat": 22.5600, "lon": 114.0800, "poi_id": "b2", "type": ""},
    ],
    "品牌C": [
        {"name": "C1", "address": "addr", "lat": 22.5402, "lon": 114.0602, "poi_id": "c1", "type": ""},
    ]
}
# A1-B1-C1 应该能聚合（距离极近）
# A2-B2 距离较远，不应聚合
```

| 测试用例 | 场景 | 验证 |
|----------|------|------|
| 全品牌聚合 | 所有品牌都有近距离门店 | 返回包含全部品牌的商圈 |
| 部分品牌回退 | 全品牌无法聚合 | 返回最大品牌子集 |
| 必选品牌回退 | 指定必选 + 部分回退 | 结果含所有必选品牌 |
| 单品牌 | 只有一个品牌 | 每个门店一个商圈 |
| 无门店 | 所有品牌都无门店 | 返回空列表 |
| 暴力 vs 优化 | 同一输入 | 两种算法结果一致 |

### 4.2 amap_api Mock 测试

使用 `unittest.mock.patch` 模拟 `requests.get`：

| 测试用例 | Mock 响应 | 验证 |
|----------|-----------|------|
| 正常搜索 | 标准 POI 响应 | 正确解析门店 |
| 分页 | 多页数据 | 自动获取所有页 |

## 9. CI 集成建议

```yaml
# .github/workflows/test.yml
name: Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: pip install pytest pytest-cov
      - run: pytest tests/ --cov=. --cov-report=term-missing
```
