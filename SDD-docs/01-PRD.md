## 1. 产品概述

商圈查找工具是一款帮助用户查找多个品牌门店聚集区域（商圈）的工具。用户输入目标城市和多个品牌名称，系统自动搜索各品牌在该城市的所有门店，然后通过空间聚合算法找出这些品牌门店都在指定距离范围内的区域，并以列表和地图形式展示结果。

### 1.1 目标用户

- 商业选址分析人员
- 品牌运营管理者
- 商圈调研人员
- 个人消费者（希望找到多个喜欢品牌都在附近的区域）

### 1.2 核心价值

- 自动化完成多品牌门店位置交叉分析，替代人工逐一查询
- 可视化展示商圈分布，直观了解品牌聚集情况
- 支持多种使用方式（命令行、网页、桌面插件），适配不同使用场景

## 3. 用户故事

### US-01：基础商圈搜索
> 作为一个商业分析人员，我希望输入"深圳"和"优衣库, 海底捞, 星巴克"，系统能自动找出这三个品牌门店都在 200 米内的区域，以便我分析商圈分布。

### US-02：必选品牌搜索
> 作为用户，我希望将"海底捞"标记为必选品牌，这样即使不是所有品牌都能聚集，系统也会保证结果中一定包含海底捞。

### US-03：地图查看
> 作为用户，我希望在地图上直观查看各个商圈的位置，点击标记可以看到门店名称和地址。

### US-04：CLI 批量处理
> 作为开发者，我希望通过命令行运行搜索任务，将结果导出为 JSON 文件用于后续数据分析。

### US-05：桌面快捷搜索
> 作为 uTools 用户，我希望通过快捷键呼出商圈查找，直接在桌面端完成搜索。

## 4. 功能性需求

### 4.1 品牌门店搜索 (F-01)

- **输入**：城市名称（字符串）、品牌名称（字符串）
- **处理**：
  - 调用高德地图 POI 搜索 API (`/v3/place/text`)
  - 自动分页获取所有结果（每页 20 条，最多 10 页）
  - 遇到 API 限流自动等待重试（最多 3 次，递增延迟）
  - 对搜索结果按距离阈值去重（默认 200 米内视为同一门店）
- **输出**：门店列表，每个门店包含 `name`、`address`、`lat`、`lon`、`poi_id`、`type`

### 4.2 商圈聚合 (F-02)

- **输入**：品牌-门店字典、距离阈值（米）、可选必选品牌列表
- **处理**：
  - 优先使用优化算法（空间网格索引 + 候选集剪枝）
  - 检查所有门店组合中两两距离是否均在阈值内
  - 若无全品牌聚集，自动从多到少枚举品牌子集（≥2 个品牌）
  - 若指定了必选品牌，子集必须包含所有必选品牌
  - 对结果去重：每个门店只归属品牌数最多的商圈
- **输出**：商圈列表，每个商圈包含 `brands`、`stores`、`max_distance`、`brand_count`

### 4.3 Web 搜索 (F-05)

- **登录**：用户名密码认证，session 管理
- **搜索**：表单提交后通过 SSE 流式推送进度
- **进度**：显示搜索阶段、当前品牌、进度百分比、详细日志
- **结果**：左侧地图 + 右侧商圈列表侧边栏
- **交互**：点击商圈卡片跳转地图位置，点击标记显示详情

### 4.4 必选品牌 (F-07)

- **前端**：品牌输入框下方实时渲染品牌标签，点击切换必选状态
- **后端**：验证必选品牌是搜索品牌的子集
- **算法**：部分品牌回退时过滤不含必选品牌的子集
- **兼容**：不指定必选品牌时行为与之前完全一致

## 5. 非功能性需求

### 5.1 性能

- 优化算法应将组合检查数量减少 90% 以上（对比暴力枚举）
- Web 端搜索过程实时推送进度，避免用户等待无反馈
- 生产环境支持 gunicorn 多 worker 部署

### 5.2 安全

- Web 端需登录后才能使用
- API 密钥通过环境变量配置，不硬编码
- 生产环境应更换默认的 `SECRET_KEY` 和登录凭据

### 5.3 兼容性

- Python 3.7+
- 主流浏览器（Chrome、Firefox、Safari、Edge）
- 移动端响应式适配（768px 断点）

### 5.4 可用性

- 搜索参数自动保存，刷新页面不丢失
- 中文逗号自动转换为英文逗号
- 距离阈值范围校验（50-5000 米）

## 7. 术语表

| 术语 | 定义 |
|------|------|
| 商圈 | 多个品牌门店两两距离均在阈值范围内的聚集区域 |
| 必选品牌 | 用户指定的在部分匹配回退时必须包含的品牌 |
| 距离阈值 | 判断门店是否在同一商圈的最大距离（米） |
| POI | Point of Interest，兴趣点（高德地图数据概念） |
| SSE | Server-Sent Events，服务端推送事件流 |
| 门店去重 | 将距离极近的同品牌门店视为同一门店并合并 |
| 部分品牌回退 | 当无法找到全部品牌聚集时，自动降级搜索品牌子集 |
